git基本指令

mkdir name 創建一個文檔夾

rm file 刪除

cd name 進入文檔夾

pwd 查看當前目錄

git init 把當前目錄變成git可以管理的倉庫 生成.git文檔夾

git add name 把文檔添加到暫存區stage

git commit -m “ ” 把暫存區所有文檔提交到git當前分支上（本地倉庫）

git status 查看當前git倉庫狀態

git diff file 查看修改的文檔內容

git log 查看從最近到最遠的提交日誌

git reset --hard 版本號  回退到某版本（HEAD指向當前版本）

git push origin HEAD --force 上傳目前(hard)版本

git reflog  查看命令操作歷史

git checkout -- file    丟棄工作區文檔的修改

git reset HEAD file    丟棄暫存區的文檔

git rm file  刪除工作區文檔後還需刪除git中的文檔

git checkout --file  恢復誤刪的git文檔

關聯一個遠程倉庫   git remote add origin address

第一次推送代碼到遠程倉庫 git push -u origin master

（-u會把本地倉庫master分支和遠程倉庫的master分支關聯起來）

從遠程倉庫克隆代碼到本地 git clone address

創建分支 git branch name

切換分支 git checkout name     

查看分支 git branch

創建分支並切換到分支上　　git checkout -b name

合併指定分支到當前分支上 git merge name

合併後刪除分支 git branch -d name

合併時出現衝突要手動解決衝突，再提交，最後合併。

將寫到一半的代碼從工作區拿走存起來 git stash

恢復隱藏的代碼到工作區並刪除隱藏位置的代碼  git stash pop

Ggame-synopsis

介紹

Ggame 並不是一個功能齊全的遊戲API，每一個鈴鐺和哨子。它主要是作為教授計算機編程的工具而設計的，認識到創建引人入勝的互動遊戲的能力是許多計劃學生的強大動力。因此，可以由用戶合理地實現的任何功能或性能增強被留作練習。

功能目標

ggame庫旨在簡單易用。例如：

"""
Trivial example of a minimal ggame App with Sprite.
"""
from ggame import App, ImageAsset, Sprite

# Create a displayed object at 100,100 using an image asset
Sprite(ImageAsset("bunny.png"), (100, 100))
# Create the app, with a default stage
APP = App()
# Run the app
APP.run()
擴展

Ggame正在以一種讓人聯想到Geogebra，數字邏輯模擬以及用於火箭和軌道模擬的工具和類的方式進行幾何探索。

概述

ggame應用程序中有三個主要部分：資產，精靈和應用程序本身。

資產

資產對象（即ImageAsset等）通常表示由“藝術部門”提供的單獨文件。這些可能是背景圖像，用戶界面圖像或代表遊戲中對象的圖像。此外，SoundAsset 還用於表示可在遊戲中播放的聲音文件（.wav或.mp3格式）。

Ggame還擴展了資產概念，以包括在運行時動態生成的圖形，例如幾何對象，例如矩形，線條等。

精靈

遊戲的所有視覺方面都由它的實例Sprite或子類表示。

應用

每個ggame應用程序都必須創建一個App類的實例（或它的子類）。創建App類的實例以在瀏覽器窗口中繪製圖形畫布。執行應用程序的run()方法以開始刷新並重新繪製屏幕上的可視資源。

事件

沒有玩家和玩家製作活動，遊戲就不會完整。您的代碼通過使用類listenKeyEvent()和listenMouseEvent()方法註冊接收鍵盤和鼠標事件來處理用戶輸入 App。

執行環境

Ggame旨在使用Brython， Pixi.js和Buzz在Web瀏覽器中執行。最簡單的方法是從runpython執行 ，源代碼存儲在 github上。當您在runpython中使用ggame時 ，Github ggame存儲庫將自動放置在導入搜索路徑中。

幾何

當提到屏幕坐標時，請注意計算機屏幕的x軸是水平的，屏幕左側的零位置。y軸是垂直的，屏幕頂部的零位置。

增加正y坐標對應於計算機屏幕上的向下方向。請注意，這與您在數學課程中學習x和y坐標的方式不同！

class ggame.app.App（* args ）[source]

的App類是一個封裝處理顯示系統的，和處理用戶事件（通常子類）的類。該App 班還負責管理所有的列表Sprite中的應用實例。

在子類App化時，您可以選擇在初始化部分中實例化大多數精靈對象。

的App類是通過指定以像素為單位所希望的應用程序窗口大小，作為兩個實例化參數要么::

myapp = App(640,480)

或者根本不提供尺寸參數::

myapp = App()

在這種情況下，使用完整的瀏覽器窗口大小。

注意：一次只能實例化一個類或子類的一個實例App。

Frame是一個實用類，用於表達矩形區域的概念。

初始化參數描述框架左上角的位置以及框架的寬度和高度。單位通常 以像素為單位，但幀可以是通用的。

  • y（int） - 框架左上角的Y坐標
  • w（int） - 幀的寬度
  • h（int） - 框架的高度
x

y

該框架左上角的Y坐標。

w

框架的寬度。

h

框架的高度。

精靈
類ggame.sprite.Sprite（資產，POS =（0，0） ，edgedef =無）[源]

Sprite類結合了可視/圖形資產的概念，屏幕上的位置和行為。雖然Sprite可以按原樣使用，但它通常是子類，以給它一些所需的行為。

在子類化Sprite類時，您可以自定義初始化代碼以使用特定資產。可以添加“步驟”或“輪詢”方法來處理每幀動作（例如，檢查衝突）。App類不會自動調用步或輪詢函數，但您可以為App該類創建子類。

此外，您可能希望在自定義的sprite類中定義事件回調方法。通過自定義創建，事件處理和定期處理，您可以實現精靈對象的完全自主行為。

參數：
  • asset（asset） - 現有的圖形資產
  • pos（tuple （int ，int ）） - 可以提供精靈位置，它指定屏幕上精靈的起始（x，y）坐標。默認情況下，精靈的位置定義其左上角的位置。可以通過自定義此行為來修改此行為 center。
  • edgedef（資產） - 可以提供邊緣定義資產，其指定將用於定義精靈的邊界以用於衝突檢測的資產。如果沒有 給出edgedef資產，則使用所需資產，在圖像紋理的情況下，該資產將是矩形資產。此選項通常用於為具有透明紋理圖像背景的基於紋理的精靈定義可見圖像輪廓。
返回：

沒有。如果位置在屏幕上，精靈將顯示在瀏覽器中。

例：

from ggame.sprite import Sprite
from ggame.asset import ImageAsset, CircleAsset
from ggame.app import App

player = Sprite(
    ImageAsset("bunny.png",
    (100,100),
    CircleAsset(50))

App().run()

這將使用'player.png'圖像創建一個精靈，該圖像的左上角位於坐標（100,100）處，並且具有50像素半徑的圓形碰撞邊界。
