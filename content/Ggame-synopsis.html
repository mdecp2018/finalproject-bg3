Ggame-synopsis

介紹

Ggame 並不是一個功能齊全的遊戲API，每一個鈴鐺和哨子。它主要是作為教授計算機編程的工具而設計的，認識到創建引人入勝的互動遊戲的能力是許多計劃學生的強大動力。因此，可以由用戶合理地實現的任何功能或性能增強被留作練習。

功能目標

ggame庫旨在簡單易用。例如：

"""
Trivial example of a minimal ggame App with Sprite.
"""
from ggame import App, ImageAsset, Sprite

# Create a displayed object at 100,100 using an image asset
Sprite(ImageAsset("bunny.png"), (100, 100))
# Create the app, with a default stage
APP = App()
# Run the app
APP.run()
擴展

Ggame正在以一種讓人聯想到Geogebra，數字邏輯模擬以及用於火箭和軌道模擬的工具和類的方式進行幾何探索。

概述

ggame應用程序中有三個主要部分：資產，精靈和應用程序本身。

資產

資產對象（即ImageAsset等）通常表示由“藝術部門”提供的單獨文件。這些可能是背景圖像，用戶界面圖像或代表遊戲中對象的圖像。此外，SoundAsset 還用於表示可在遊戲中播放的聲音文件（.wav或.mp3格式）。

Ggame還擴展了資產概念，以包括在運行時動態生成的圖形，例如幾何對象，例如矩形，線條等。

精靈

遊戲的所有視覺方面都由它的實例Sprite或子類表示。

應用

每個ggame應用程序都必須創建一個App類的實例（或它的子類）。創建App類的實例以在瀏覽器窗口中繪製圖形畫布。執行應用程序的run()方法以開始刷新並重新繪製屏幕上的可視資源。

事件

沒有玩家和玩家製作活動，遊戲就不會完整。您的代碼通過使用類listenKeyEvent()和listenMouseEvent()方法註冊接收鍵盤和鼠標事件來處理用戶輸入 App。

執行環境

Ggame旨在使用Brython， Pixi.js和Buzz在Web瀏覽器中執行。最簡單的方法是從runpython執行 ，源代碼存儲在 github上。當您在runpython中使用ggame時 ，Github ggame存儲庫將自動放置在導入搜索路徑中。

幾何

當提到屏幕坐標時，請注意計算機屏幕的x軸是水平的，屏幕左側的零位置。y軸是垂直的，屏幕頂部的零位置。

增加正y坐標對應於計算機屏幕上的向下方向。請注意，這與您在數學課程中學習x和y坐標的方式不同！
